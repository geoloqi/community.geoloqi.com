    Hi there, I tried with a fresh download of the SDK from https://github.com/geoloqi/Geoloqi-iPhone-SDK, and wrote the same method you had above trying to print the userID from the LQSession object. I didn't have any trouble, in my XCode logs I get the expected result:

    The only thing I can think of is to make sure you've downloaded the latest SDK from Github and make sure you do a fresh clean and build.

    You should also note that you don't need the user ID on the phone most of the time. Any time you're dealing with methods like layer/subscribe/:id, the authenticating user will be subscribed to the layer so you don't actually need to specify the user ID in the request.
