    Hey There! You can use the visual layer editor on the website to create a layer with several places. Those places can have a trigger associated to them that will fire some action when the user enters or exits that region. You then just need to subscribe the user to that layer using the Geoloqi REST API. You can use the Android SDK to make calls to the API. Check out this tutorial:

    https://developers.geoloqi.com/android/making-api-requests

    Here are the API methods you'll be most interested in:

    https://developers.geoloqi.com/api/layer/list
    https://developers.geoloqi.com/api/layer/subscribe/:id
    https://developers.geoloqi.com/api/geonote/create
    https://developers.geoloqi.com/api/trigger/create

    Let us know if you have any more questions!

    -Tristan

    If you're just creating a message trigger (geonote) the SDK will handle displaying the message for you automatically. You only need to implement the onPushMessageReceived callback if you want to do something different when a push is received.

    Unfortunately, we haven't yet released a version of the Android SDK with full push messaging support. It'll be released this weekend and should resolve your issues.
