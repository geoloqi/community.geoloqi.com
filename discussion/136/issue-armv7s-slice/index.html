    We will be releasing a new version of the SDK with armv7s support soon. In the meantime, you can get around this error by removing 'armv7s' from the list of Valid Architectures in your target's Build Settings. You will still be able to run apps that are built for armv7 on an iPhone 5 just fine.  Hope this helps.
